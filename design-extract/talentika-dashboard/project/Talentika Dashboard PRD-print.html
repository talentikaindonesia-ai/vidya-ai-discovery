01 — Overview

What Talentika is, and why this PRD exists

This document is the single source of truth for Claude Code to implement the Talentika dashboard. It locks the brand system, the screens, the tokens, and the acceptance criteria so the build matches the design.

Product Vision

Help every young person in Indonesia discover their unique potential, develop it with structured guidance, and grow through opportunities and community.

Brand Personality
★ Inspiring ⚡ Empowering ♥ Supportive ◆ Trustworthy ✦ Innovative

Tone of voice: Positif & Memotivasi · Ramah & Dekat · Jelas & Informatif · Membangun & Solutif.

Scope of this PRD

Covers the post-login member dashboard for the Individual user type (students & young professionals). Membership/Pro upsell surfaces are referenced but the full billing flow is out-of-scope for v1.

02 — Users

Who we're building for

Three primary personas drive the dashboard's structure. Every flow should serve at least one persona's "job to be done".

D

Muhammad Dafa — “The Explorer”

SMA / Early university · 16–19 yrs

Curious but unsure which path fits. Wants to “try things” without commitment, and to see proof his interests can lead somewhere.

  • JTBD: “Help me figure out what I'm good at, and what to do next.”
  • Cares about: assessment results, beginner courses, scholarships, peer community.
S

Siti — “The Climber”

University · 19–22 yrs

Has a direction (e.g. UI/UX) and wants structured growth: a roadmap, real challenges, a portfolio of badges.

  • JTBD: “Give me a clear path and prove I'm progressing.”
  • Cares about: roadmap, progress %, challenges, internships, certificates.
R

Rian — “The Achiever”

Fresh grad / Early career · 22–25 yrs

Goal-oriented; treating Talentika like a launchpad. Wants premium content, mentorship, and high-signal opportunities.

  • JTBD: “Connect me to the next real step in my career.”
  • Cares about: Pro upgrade, conferences, jobs, mentorship, community leadership.

07 — Screens

Screen-by-screen spec

Each screen lists route, purpose, layout regions, data, primary actions, and acceptance criteria. Image references are 1:1 with the brand mocks.

7.1 — Authentication

/auth · /auth/register
Public Tabs: Masuk · Daftar Akun Locale: ID

Layout

Two‑column split. Left = brand panel (gradient blob illustration + value props + hero copy "Mulai perjalanan terbaikmu bersama Talentika"). Right = card with tab switcher, form, social auth.

Form fields

  • Email atau Nomor Handphone (smart input — detects @ vs digits)
  • Kata Sandi with eye toggle
  • Forgot password link, right-aligned
  • Sticky CTA: "Masuk →" full width, blue 600
  • OAuth: Google · Apple · Facebook (equal-weight, monochrome)

Value props (left rail)

  • Akses Pembelajaran Berkualitas — blue tint icon
  • Komunitas Positif & Inspiratif — orange tint icon
  • Bangun Masa Depanmu — mint tint icon

Acceptance

  • Form validates inline (no full-page reload)
  • Footer microcopy links to Syarat & Ketentuan and Kebijakan Privasi
  • "Belum punya akun? Daftar sekarang" swaps tab without route change

7.2 — Onboarding (5 steps)

/onboarding/[1..5]
First‑run only Lewati allowed Persistent dot pagination

Steps

  1. Kenali dirimu lebih dalam — intro to Talent Test (illustration: youth with laptop).
  2. Dapatkan insight karier yang tepat — recommendations preview.
  3. Bangun roadmap pengembangan diri — mountain path illustration.
  4. Bergabung dengan komunitas positif — group of friends.
  5. Siap untuk masa depanmu? — rocket; CTA Mulai Sekarang.

Layout rules

  • Desktop: 5 cards in a row preview; Mobile: one card at a time, swipe.
  • Each card: illustration block (60% height) over title + body.
  • Footer strip with 3 trust signals: Aman & Terpercaya · Privasi Terjaga · Berdasarkan Sains.
  • "Lewati" persists in top-right across all steps.

Acceptance

  • Final step CTA routes to /talent-test not dashboard.
  • Onboarding state stored in user profile → never shown again.

7.3 — Dashboard (Home)

/dashboard
Authenticated · Default landing 3 main regions

Regions

  1. Greeting hero — full-width gradient banner: "Selamat Malam, {firstName}! 👋" + waving illustration on right.
  2. Hasil Assessment Anda — avatar circle, persona ("Artistic (Creator)"), 1-line summary, "Tingkat Kesesuaian" progress, recommended career chips, Area Bakat Utama list, Tes Ulang / Lihat Kursus actions.
  3. Right column — Ringkasan Progress (4 mini-stats grid) + Kursus Anda (top 3 with progress + chevron) + Aktivitas Terbaru horizontal feed.

Data & behavior

  • Greeting time-aware (Pagi/Siang/Sore/Malam).
  • Notification bell shows live count badge (top-right).
  • Theme toggle next to bell — light/dark, no system flicker.
  • Avatar menu reveals profile + sign-out + plan switch.
  • "Tes Ulang" opens confirmation modal (retesting cools down 30 days).

Acceptance

  • If user hasn't taken Talent Test → block #2 shows empty state CTA "Mulai Talent Test".
  • Activity feed shows last 4 events with relative timestamps in ID locale.

7.4 — Kursus Saya

/courses
Persistent main nav Filter: Semua Status ▾

Layout

  • Header row: page title + subtitle + status filter (right).
  • 4 stat cards: Kursus Aktif · Jam Belajar · Progress Rata‑rata (radial) · Sertifikat.
  • Main panel: Kursus Aktif list (5/page) with course rows + pagination.
  • Right rail: Kalender Belajar (month view) → Rekomendasi Untukmu (paid placement allowed) → Pencapaian Terbaru.

Acceptance

  • Course row click opens course detail (out of v1 scope, stub route).
  • Progress bar color stays blue regardless of %; never red.
  • Calendar marks today + every day with scheduled study session.
  • Recommendation card uses radial accent if "sponsored".
  • Pagination: "Menampilkan 1 – 5 dari N kursus" stays in lower-left.

7.5 — Peluang (Opportunities)

/opportunities
Discovery Tabs: Semua · Beasiswa · Kompetisi · Magang · Lowongan · Konferensi Saved state

Layout

  • Top hero: centered title "🎁 Opportunity for You" + subtitle + decorative graduation cap & trophy SVGs on right.
  • Category tab row, each with count chip.
  • 3-col grid of opportunity cards (2 rows shown, paginated).
  • Selected/hovered card flips banner to filled blue + CTA flips to filled.

Card anatomy

  • Banner with category pill (top-left) + source domain (centered) + bookmark icon (top-right, fills when saved).
  • Title 2 lines max with line-clamp; "Internasional" location row.
  • 2–3 tags (scholarship/phd/sosial).
  • Footer CTA "Lihat Detail →".

Acceptance

  • Filter tab updates count + URL query.
  • Saved items persist server-side (per-user).

7.6 — Community

/community
Discovery + upsellPro membership CTA

Layout

  • Top hero: "👥 Community for You" with decorative chat/heart SVGs.
  • Carousel of 3 recommendation cards per page (dot pagination).
  • Each card: category pill (top-left, tonal) + Rekomendasi pill (top-right, lilac), title, 2-line description, member & discussion count, tag pills, primary CTA "Bergabung" (active card filled blue tint).
  • Below: full-width "Explore All Communities →" outline CTA.
  • Membership banner: illustration + headline + "Akses Membership Dashboard →".

States

  • Pro CTA only shows for non-Pro users.

08 — Motion & Interaction

How Talentika moves

Motion direction: smooth, dinamis, dan penuh energi positif. Never bouncy-cartoonish; never sterile-flat.

TokenCurveDurationUse
ease-softcubic-bezier(.2,.7,.2,1)220msDefault UI transitions
ease-liftcubic-bezier(.16,1,.3,1)320msCard hover-lift, modal in
ease-snapcubic-bezier(.4,.0,.2,1)160msToggle, tab switch
page-infade + translateY(8px)280msRoute transitions
sparklescale 0→1 + rotate ±15°600msAchievement unlock

Microinteractions to include

Greeting hero waving hand loops on mount. Course progress bar fills with 600ms ease-lift on first paint. Bookmark icon morphs (outline → filled) with a 200ms scale pulse. Sparkles (★) drift slowly in onboarding step backgrounds. Reduce motion preference disables all but opacity transitions.

Accessibility

  • WCAG AA contrast for all text + interactive UI.
  • Focus rings: outline: 2px solid var(--blue-600); outline-offset: 2px.
  • All icons paired with labels or aria-label.
  • Keyboard navigable nav: Cmd/Ctrl + K opens search; arrow keys traverse cards.
  • Respect prefers-reduced-motion.
  • Color is never the only signal (use icons + text).

11 — Build Order

Suggested build order for Claude Code

Implement in this sequence so each layer can be reviewed before the next is built on top.

  1. Tokens & theme. Add Poppins/Inter, write styles/tokens.css, wire Tailwind theme + dark theme variables.
  2. Primitives. Button, Input, Card, Pill, Avatar, Progress, Tabs, Modal, Toast.
  3. Layout shell. Sidebar (with collapse), Topbar (search/theme/notif/avatar), AppShell, AuthLayout.
  4. Auth + Onboarding. Static screens first, then mock auth, then real auth.
  5. Dashboard skeleton. Greeting hero, assessment card, progress + courses + activity. Mock API.
  6. Kursus Saya. Stat row + course list + right rail (calendar, recs, pencapaian).
  7. Peluang + Community. Card grids, filters, save state, carousel.
  8. Remaining surfaces. Progress, Pencapaian, Timeline, Profile, Pengaturan.
  9. Motion pass. Apply ease tokens to all hover/route transitions, sparkle and waving microinteractions.
  10. A11y + i18n + perf pass. Run axe, finalize ID strings, measure LCP, ship.

Out of scope for v1

Live mentorship video, payment processing, admin/CMS dashboard, full Talent Test engine logic (UI shell only), notification email templates, mobile-native apps.
